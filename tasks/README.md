# tasks

Execution-ready artifacts derived from idea and PRD workflows.

## Responsibility

- Hold planning artifacts that drive implementation execution.
- Track dependency order and status for vertical slices.
- Keep task granularity aligned with user stories and acceptance criteria.

## Runtime Mapping

In a real project, this folder maps to:

`tasks/ -> .agents/tasks/`

## Typical Outputs

- `qa-ideas.md` from ideation (`rb-idea`)
- `qa-requirements.md` from PRD generation (`rb-prd`)
- `task-index.md` dependency/status index
- `task-*.md` slices generated by `rb-break-task`
