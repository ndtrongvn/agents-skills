# Contributing

Thanks for contributing to `agents-skills`.

## Scope

This repository stores reusable skill definitions. Keep contributions focused on:

- skill behavior clarity
- deterministic contracts
- portability across AI coding agents
- safety and anti-hallucination quality

## Skill Contract Standards

Every skill folder should include:

- `SKILL.md` with YAML frontmatter (`name`, `description`)
- clear objective and execution flow
- explicit required inputs and outputs
- deterministic error messages for missing prerequisites

When adding templates/scripts:

- keep template sources inside the skill folder
- avoid hardcoding project-specific assumptions
- document how runtime paths map to `docs/...` chain artifacts

## Writing Guidelines

- Use concise, imperative instructions.
- Prefer file-gated contracts over implicit conversational state.
- Define failure modes with exact error strings.
- Avoid ambiguous language like "maybe", "probably", or "usually" for required behavior.

## Path and Runtime Conventions

- This repository contains reusable assets.
- Runtime artifacts generated by skills belong in this repository under:
  - `docs/` (chain outputs)
  - root `AGENTS.md` (when relevant)

Do not commit generated runtime artifacts unless intentionally adding examples.

## Quality Checklist

Before opening a PR:

1. Validate every referenced path in `SKILL.md` exists or is intentionally runtime-generated.
2. Ensure templates render valid markdown with no broken placeholders.
3. Ensure deterministic output contracts remain backward compatible, or document breaking changes.
4. Confirm README/guides tables mention any new skill or changed behavior.

## Commit and PR Guidelines

- Keep PRs small and focused.
- Use descriptive commit messages.
- In PR description include:
  - what changed
  - why it changed
  - migration notes (if contract changed)
  - example invocation or output

## Security

If a skill change affects secrets, auth boundaries, or data safety rules, call it out explicitly in the PR and request a security-minded review.
